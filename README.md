# 数据中台系统
#### 介绍
公司有整套商业版数据中台系统源码出售， **需要演示环境和方案PPT，可加V：gzzw1131** 

数据中台集成元数据管理、数据源整合、数据治理、数据标准定义、数据标签体系、数据服务化、数据质量监控、数据资产管理、数据开发采集及数据可视化等全链路功能，旨在为企业提供一站式数据管理与价值挖掘解决方案。

#### 功能清单

##### 系统管理
- 基础管理-用户管理	支持用户的维护、重置用户密码、为用户设置角色、部门、职位。
- 基础管理-角色管理	维护系统中的角色，支持为角色进行功能模块授权
- 基础管理-部门管理	维护公司的组织机构
- 基础管理-职位管理	维护公司的职位信息
- 基础管理-定时周期管理	维护crontab表达式，为系统中的定时执行任务提供crontab表达式下拉选项
- 开发管理-字典管理	管理系统中的数据字典，可以为系统中的部分下拉框动态添加修改可选项
- 开发管理-菜单管理	管理系统中的菜单
- 平台审计-登录日志	支持查看所有中台用户的登录日志，包含时间、操作系统、ip、浏览器
- 平台审计-操作日志	支持查看所有中台用户的操作，用于审计
- 系统日志-实时日志	查看各个微服务正在打印的日志，滚动刷新，支持选择单个微服务和单个微服务实例过滤。方便数据开发人员及时掌握数据引擎打印的实时日志。
- 系统日志-日志查询	可以查询历史系统日志，支持根据时间、微服务名称、日志级别、追踪码进行日志过滤，方便系统维护人员维护系统。
- 数据运维-数据服务	数据服务监控以图表的形式展示数据服务调用情况和异常情况以及性能情况，帮助实施人员优化数据服务响应速度、解决数据服务问题。
- 服务调用日志  支持查看不同的应用调用服务情况，支持查看详细调用日志和syslog。"
- 数据运维-ITMS	服务器支持监控部署数据中台系统的各个服务器CPU、内存、硬盘、带宽、操作系统等信息。数据库可以维护MySQL和redis类型的数据库信息，支持监控数据仓库的在线状态、业务库在线状态、redis状态"
- 数据运维-数仓备份	数仓备份可以维护远程仓库信息，可以通过定时自动备份将数仓中的数据备份到远程仓库中，支持将备份的远程仓库的数据恢复到系统数仓中。

##### 数据建设平台
- 元数据-数据源管理	支持将各个业务系统的数据源连接参数进行配置，方便后面做数据采集使用。
- 元数据-元模型	元模型是用于存储元数据的数据模型，系统支持动态维护元模型，当元数据结构发生改变的时候无需二次开发即可适配。
- 元数据-最新元数据	"指的是最新从文件、接口、数据库采集过来的元数据，在最新元数据中支持手动添加和配置元数据之间的关联关系。
- 支持通过元数据的数据血缘分析，可以分析出一个数据模型的数据是从哪个系统来的，在数据治理过程中经过了哪些数据模型流向到目标模型。
- 支持通过元数据的影响分析，可以查看当一个表字段、表发生变动(如删除或者改名)会对哪些数据模型产生影响。"
- 元数据-元数据同步	通过定时任务自动采集各种数据源的元数据，包含csv,excel,jdbc等采集适配支持
- 元数据-元数据审核	当开始进行数据接入时，我们将当前元数据进行定版提交和审核。发布了版本后方便与最新元数据进行对比找出差异后调整数据采集程序。
- 元数据-定版元数据	元数据提交定版申请审批通过后形成的确定版本的元数据，系统支持多哥元数据版本。
- 元数据-元数据分析	元数据对比可以选择2个不同的元数据版本进行对比，会显示各个元模型下的元数据的新增，修改，删除等详细信息。方便维护人员及时调整数据采集任务。
- 数据标准-标准定义	定义数据仓库的设计标准，包括模型设计标准，数据元标准，字典标准，标准词根，业务术语
- 数据标准-发布管理	发布内部标准到发布目录的申请
- 数据标准-审核管理	审核数据标准发布请求
- 数据标准-数据标准核对	对数据仓库的数据进行标准核对，查看数据标准执行情况

##### 数据治理平台
- 主数据-主数据实体	维护主数据字段，字段数据编码规则，主数据字段内容加密规则，主数据版本，组合唯一键等信息
- 主数据-编码规则	维护主数据的数据内容生成规则信息
- 主数据-主数据目录	查看主数据具体数据，数据不同版本之间的差异对比，数据导入
- 数据仓库-doris集群管理	维护数仓连接信息和需要的数据库信息，支持动态接入多个数据仓库集群
- 数据仓库-数仓规划	维护数仓主题域、业务主题和数仓分层信息，数据分层可以根据数据模型的使用场景进行宏观分层，方便管理查找数据模型。
- 数据模型-模型数据	可以查看数据模型不同数据表中的数据，能够对数据进行筛选并将数据导出成SQL、CSV和EXCEL文件
- 数据模型-维度建模	在线设计存储数据的数据模型，支持主键模型、明细模型和聚合模型。
- 数据模型-模型运维	支持为已经存在的数据模型提交新增字段申请以及审批。
- 数据指标-指标开发	在线开发原子指标和复合指标
- 数据指标-指标地图	查看发布后的正式版本的数据指标，支持指标数据以列表、柱状图、折线图展示
- 数据质量-规则管理	通过系统内置的规则模板配置模型数据质量规则，比如哪些字段不能重复、不能为空等
- 数据质量-检测任务	使用数据质量规则来对数据仓库的数据进行数据质量评估、生成数据质量评估报告。
- 数据标签-标签实体	给不同的数据对象打标签，比如订单、合同都可以称之为数据实体
- 数据标签-标签管理	为维护实体下的标签分类和标签比如 学生的性别、年龄、籍贯、爱好、专业、家庭收入等都可以称之为标签。
- 数据标签-置标任务	使用定时任务的方式对各个标签进行标签值计算
- 数据标签-标签圈群	使用多个数据标签对对象进行群体划分，如：学习好并且喜欢打篮球的男生
- 数据标签-圈群结果	使用标签群体对标签实体的数据进行过滤，支持过滤结果的分页查看和导出
- 数据标签-标签画像	通过数据标签来分析每一个标签中各类数据的比例和数量。支持饼图和柱状图显示。
- 数据安全-分级分类	"使用分级和分类来控制数据权限，比如绝密、机密、秘密三个分级，有绝密权限的人可以看到机密和秘密的数据。有秘密级别的人只能看到秘密级别的数据。
- 分类用于对整体数据进行类别划分，只有拥有指定分类数据权限的人才能看对应分类的数据。"
- 数据安全-数据脱敏	对部分数据模型的部分字段进行脱敏配置，配置完成后在线查看数据的时候只能看到脱敏后的数据。

##### 数据服务平台
- 服务管理-服务开发	支持通过简单配置的方式来开发一个数据接口、支持通过SQL+JS代码的方式开发数据接口、支持将自己通过代码写的主接口发布到中台统一管理
- 服务管理-服务测试	在线开发完的接口可以在此功能完成接口测试，并且可以将测试用的输入输出保存为接口调用示例，方便生成文档的时候为其他开发者参考。
- 服务管理-API集市	服务开发并审核完成的接口可以在API集市中查看，可以为具体的应用申请服务调用权限
- 服务管理-应用管理	管理需要调用数据中台接口的第三方系统账号，会为每个系统分配appId和appKey来做认证。
- 审核管理-服务审核	包含接口对外发布的审核以及数据应用需要调用此接口而提交的使用申请的审核
- 审核管理-服务申请	查看自己申请的服务发布、应用权限、其他服务权限信息，可以对已经申请的审核进行撤销
- 审核管理-历史审核	展示已经申请审核过的服务发布、应用权限、其他服务权限信息
- 数据资产-资产标签	维护资产标签信息
- 数据资产-目录管理	包含数据资产标签的管理、多版本的数据目录编目功能。本模块主要为了将中台中的指标、报表、接口、数据标签、数据文件、元数据等对外统一发布使用。在资产编目的时候一般会根据数据资产使用者的习惯进行分类。
- 数据资产-资产编目	支持将系统中的数据模型、元数据、数据标签实体、报表、数据指标等进行编目，将编目好的资产目录发布后可以在资产门户中进行展示。
- 数据资产-资产门户	支持展示资产数量，查找收藏使用各类的数据资产，包含报表、指标、数据文件、数据接口、元数据、数据标准、数据标签等。

##### 数据采集平台
- 离线开发-任务管理	"使用kettle作为ETL引擎，支持在线设计etl执行流程和对etl进行调度功能。
- 支持数据库、kafka、mqtt、mongo、es、http接口、webservice、ftp、sftp、stps、csv、excel、文本文件等数据的接入、治理功能。"
- 离线开发-文件资源管理	ETL需要用到的文件资源以及ETL生成的文件资源可以在文件资源进行管理，包含上传、下载、删除、文件预览等功能
- 多数据源目录	数据仓库的远程虚拟数据库，方便做高性能数据采集使用。支持多源数据目录的创建、刷新、删除等功能
- 实时开发-任务管理	底层使用Flink引擎，支持数据的实时采集(业务系统新增数据后，数据中台立刻将数据采集到数据中台中来)。支持FlinkSQL和FlinkJar 2种模式
- 实时开发-告警通知	维护用于接收告警通知的人和他的通知接收地址(邮箱、手机号)
- 实时开发-JAR维护	使用FlinkJar任务的时候，需要将JAR通过本功能上传到数据中台，在任务管理中可供选择使用。
- 数据挖掘-项目管理	对数据挖掘项目的挖掘流程定义、流程版本管理、挖掘任务的创建、任务版本管理，通过项目概览可以查看项目的流程、任务的执行状态报表
- 数据挖掘-源中心	管理数据挖掘中使用到的数据源信息，方便在创建数据挖掘任务的时候进行选择
- 数据挖掘-资源中心	管理UDF、任务过程中用到的JAR、Shell、SQL等文件
- 数据挖掘-配置中心	配置worker分组、环境变量、Yarn队列、告警等

##### 数据可视化平台
- 数据可视化-数据集管理	从数据仓库中抽取数据形成数据集，方便后面的数据组件展示使用。
- 数据可视化-数据源管理	管理数据源，用于出报表的数据源可以是数据中台的数据仓库，也可以是其他的关系型数据库
- 数据可视化-报表管理(excel类和数据大屏)	通过拖拉拽的方式实现excel报表和数据大屏报表的开发
- 数据可视化-文件管理	excel报表中包含导出功能，导出的数据在文件管理中可以下载
- 知识图谱-图谱管理	支持通过配置的方式将数据仓库中的数据导入到图库中，生成知识图谱在线查看
